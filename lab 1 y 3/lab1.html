  1. ¿Cuál es la diferencia entre Internet y la World Wide Web?

  2. ¿Cuáles son las partes de un URL?

  3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
    • HTTP: Realizar petición de datos o informacón
    • GET: Peticion para recuperar dato en especifico
    • HEAD: Manda header y hacer diagnostico
    • POST: Enviar informacion
    • PUT: Modificar todo el recurso
    • PATCH: Modifica solo una parte previamente especificada del recurso
    • DELETE: Eliminar un recurso

  4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio?

    5. GET Y POST: GET->envia los datos. POST->los envia de manera cifrada.

  5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

  6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

  • Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

    • Como recomendacion es que solo sea utilizado cuando en realidad se busque resaltar algo y dejar a un lado las reglas previas, no es bueno usar simpre el ¡important ya que se estara sobrescribiendo todo el texto dejando a un lado las reglas de estilo previamente marcadas.

  • Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

    • Debe escogerse con cuidado dado las definiciones de las pantallas, es decir, del tamaño de las ventas, al igual, los pixeles interfieren, al momento de ser muy pequeños, la pagina tiene que trabajar mucho en repetir cada uno de ells y finalmente puede afectar la estetica.

  • Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

    • Recomiendo utilizar px cuando se edita width, ya que asi el ancho sera el mismo sin importar los cambios futuros; el % utilizarlo cuando busquemos un valor relativo a otro, es decir, relativo a un padre, y finalmente los pt y px cuando se edite en cuestion de unidades absolutas, es decir, independientes,

  • ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

    • Optimiza el rendimiento del sitio ya que el mismo no tendra la necesidad de cargar y leer archivos externos al momento de agregar la estructura de la pagina, de igual manera, al comprimir los datos de CSS el navegador tarda menos en realizar la lectura y pueda aplicar de manera rapida los estilos presentes. Evita el tiempo de leer caracteres no necesarios. La lectura es más rápida.
